# Alibaba Cloud Credentials for .NET - 用户使用指南

## 1. 项目简介

Alibaba Cloud Credentials for .NET 是阿里云官方提供的 .NET 凭证管理工具库，用于为各类阿里云 SDK 和应用提供统一的身份认证凭证获取能力。

### 核心能力

- **多种凭证类型支持**：支持 AccessKey、STS Token、RAM 角色、OIDC、Bearer Token 等多种认证方式
- **自动凭证刷新**：对临时凭证（如 STS Token）提供自动刷新机制，确保凭证始终有效
- **凭证提供链**：支持按优先级从多个来源（环境变量、配置文件、实例元数据等）自动获取凭证
- **跨平台支持**：支持 .NET Framework 4.5 和 .NET Standard 2.0，可在 Windows、Linux、macOS 上运行

### 典型应用场景

- 在本地开发环境使用 AccessKey 进行调试
- 在 ECS/ECI 实例上使用 RAM 角色自动获取临时凭证
- 在 Kubernetes（ACK）环境使用 OIDC 进行服务账号认证
- 在 CI/CD 流水线中使用 RAM 角色 ARN 进行跨账号访问
- 在容器或 Serverless 环境通过 URI 从凭证服务获取临时凭证

## 2. 使用前提与环境要求

### 操作系统要求

- **Windows**：Windows 7 及以上版本，推荐 Windows 10/11 或 Windows Server 2012 及以上
- **Linux**：主流发行版（Ubuntu 16.04+, CentOS 7+, Debian 9+）
- **macOS**：macOS 10.12 及以上版本

### 运行时环境

至少需要满足以下条件之一：

- **.NET Framework**：4.5 及以上版本（仅 Windows）
- **.NET Core / .NET**：支持 .NET Standard 2.0 的任意版本（推荐 .NET Core 3.1 或 .NET 6/7/8）

### 安装依赖

通过 NuGet 安装：

```bash
# 使用 .NET CLI（推荐）
dotnet add package Aliyun.Credentials

# 或在 Visual Studio 中使用 NuGet Package Manager
Install-Package Aliyun.Credentials
```

### 系统依赖

- **网络访问**：
  - 如使用 ECS RAM 角色，需访问实例元数据服务（100.100.100.200）
  - 如使用 RAM 角色 ARN 或 OIDC，需访问阿里云 STS 服务（sts.aliyuncs.com，443 端口）
  - 如使用 URL 凭证提供者，需能访问指定的 HTTP/HTTPS 地址

- **文件系统权限**：
  - 如使用配置文件，需对 `~/.alibabacloud/credentials` 或 `~/.aliyun/config.json` 有读权限
  - 如使用 OIDC Token 文件，需对指定路径有读权限

### 阿里云账号要求

- 拥有有效的阿里云账号
- 根据使用的凭证类型，需要相应的权限：
  - **AccessKey**：拥有 AccessKey ID 和 AccessKey Secret
  - **RAM 角色**：已创建 RAM 角色并授予相应权限
  - **ECS RAM 角色**：ECS/ECI 实例已绑定 RAM 角色
  - **OIDC**：已配置 OIDC 身份提供商和 RAM 角色信任策略

## 3. 快速开始指南

### 方式一：使用 AccessKey（推荐用于本地开发）

```csharp
using Aliyun.Credentials;
using Aliyun.Credentials.Models;

class Program
{
    static void Main(string[] args)
    {
        // 创建配置
        Config config = new Config()
        {
            Type = "access_key",
            AccessKeyId = "YOUR_ACCESS_KEY_ID",
            AccessKeySecret = "YOUR_ACCESS_KEY_SECRET"
        };
        
        // 初始化客户端
        var client = new Client(config);
        
        // 获取凭证
        var credential = client.GetCredential();
        Console.WriteLine($"AccessKeyId: {credential.AccessKeyId}");
        Console.WriteLine($"Type: {credential.Type}");
    }
}
```

### 方式二：使用环境变量（推荐用于生产环境）

**设置环境变量**：

Linux/macOS：
```bash
export ALIBABA_CLOUD_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID"
export ALIBABA_CLOUD_ACCESS_KEY_SECRET="YOUR_ACCESS_KEY_SECRET"
```

Windows PowerShell：
```powershell
$env:ALIBABA_CLOUD_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID"
$env:ALIBABA_CLOUD_ACCESS_KEY_SECRET="YOUR_ACCESS_KEY_SECRET"
```

**代码**：
```csharp
// 使用默认凭证提供链，会自动从环境变量读取
var client = new Client();
var credential = client.GetCredential();
```

### 方式三：使用配置文件

**创建配置文件**（`~/.alibabacloud/credentials`）：

```ini
[default]
type = access_key
access_key_id = YOUR_ACCESS_KEY_ID
access_key_secret = YOUR_ACCESS_KEY_SECRET
```

**代码**：
```csharp
// 使用默认凭证提供链，会自动从配置文件读取
var client = new Client();
var credential = client.GetCredential();
```

### 方式四：在 ECS 上使用 RAM 角色

```csharp
using Aliyun.Credentials;
using Aliyun.Credentials.Models;

// 方式 1：明确指定角色名
Config config = new Config()
{
    Type = "ecs_ram_role",
    RoleName = "YOUR_ROLE_NAME"  // 可选，不设置会自动获取
};
var client = new Client(config);

// 方式 2：使用默认凭证提供链（ECS 上会自动检测）
var client = new Client();
var credential = client.GetCredential();
```

### 启动与验证

构建并运行：

```bash
# 构建项目
dotnet build

# 运行项目
dotnet run
```

**预期输出示例**：
```
AccessKeyId: LTAI4G***********
Type: access_key
```

## 4. 常见问题解答（FAQ）

### 4.1 凭证配置相关问题

#### Q1: 提示 "AccessKeyId must not be null or empty" 或 "AccessKeySecret must not be null or empty"

**现象**：
```
System.ArgumentNullException: AccessKeyId must not be null or empty.
Parameter name: AccessKeyId
```

**原因**：
在使用 `access_key` 或 `sts` 类型时，未提供必需的 AccessKeyId 或 AccessKeySecret 参数。代码会先检查配置对象的值，如果为空则尝试从环境变量 `ALIBABA_CLOUD_ACCESS_KEY_ID` 和 `ALIBABA_CLOUD_ACCESS_KEY_SECRET` 读取。

**解决方案**：
1. 在代码中明确设置：
   ```csharp
   config.AccessKeyId = "YOUR_ACCESS_KEY_ID";
   config.AccessKeySecret = "YOUR_ACCESS_KEY_SECRET";
   ```

2. 或设置环境变量：
   ```bash
   export ALIBABA_CLOUD_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID"
   export ALIBABA_CLOUD_ACCESS_KEY_SECRET="YOUR_ACCESS_KEY_SECRET"
   ```

#### Q2: 提示 "SecurityToken must not be null or empty"

**现象**：
```
System.ArgumentNullException: SecurityToken must not be null or empty.
Parameter name: SecurityToken
```

**原因**：
使用 `sts` 凭证类型时，必须同时提供 SecurityToken（临时安全令牌）。

**解决方案**：
```csharp
Config config = new Config()
{
    Type = "sts",
    AccessKeyId = "YOUR_TEMP_ACCESS_KEY_ID",
    AccessKeySecret = "YOUR_TEMP_ACCESS_KEY_SECRET",
    SecurityToken = "YOUR_SECURITY_TOKEN"  // 必须提供
};
```

或通过环境变量 `ALIBABA_CLOUD_SECURITY_TOKEN` 提供。

#### Q3: 提示 "Unsupported credential type option"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: Unsupported credential type option: xxx, 
support: access_key, sts, bearer, ecs_ram_role, ram_role_arn, rsa_key_pair, oidc_role_arn, credentials_uri
```

**原因**：
`Config.Type` 字段值不在支持的类型列表中，或拼写错误。

**解决方案**：
确保使用以下有效值之一：
- `access_key` - AccessKey 凭证
- `sts` - STS 临时凭证
- `bearer` - Bearer Token
- `ecs_ram_role` - ECS RAM 角色
- `ram_role_arn` - RAM 角色 ARN
- `rsa_key_pair` - RSA 密钥对（已弃用）
- `oidc_role_arn` - OIDC 角色 ARN
- `credentials_uri` - URL 凭证提供者

注意：类型名称必须小写，使用下划线分隔。

#### Q4: 提示 "BearerToken must not be null or empty"

**现象**：
使用 `bearer` 类型时提示 BearerToken 为空。

**原因**：
Bearer Token 类型主要用于呼叫中心（CCC）等特定场景，必须提供有效的 token。

**解决方案**：
```csharp
Config config = new Config()
{
    Type = "bearer",
    BearerToken = "YOUR_BEARER_TOKEN"
};
```

### 4.2 配置文件相关问题

#### Q5: 提示 "Unable to open credentials file"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: Unable to open credentials file: /home/user/.alibabacloud/credentials.
```

**原因**：
1. 配置文件不存在
2. 文件路径错误
3. 没有文件读取权限

**解决方案**：

1. **确认文件路径**：
   - Linux/macOS: `~/.alibabacloud/credentials`
   - Windows: `C:\Users\YOUR_USERNAME\.alibabacloud\credentials`

2. **创建配置文件**：
   ```bash
   # Linux/macOS
   mkdir -p ~/.alibabacloud
   cat > ~/.alibabacloud/credentials << EOF
   [default]
   type = access_key
   access_key_id = YOUR_ACCESS_KEY_ID
   access_key_secret = YOUR_ACCESS_KEY_SECRET
   EOF
   ```

3. **检查文件权限**：
   ```bash
   chmod 600 ~/.alibabacloud/credentials
   ```

4. **自定义文件路径**（通过环境变量）：
   ```bash
   export ALIBABA_CLOUD_CREDENTIALS_FILE="/path/to/your/credentials"
   ```

#### Q6: 提示 "The specified credentials file is empty"

**现象**：
环境变量 `ALIBABA_CLOUD_CREDENTIALS_FILE` 被设置为空字符串。

**解决方案**：
```bash
# 取消设置或设置为有效路径
unset ALIBABA_CLOUD_CREDENTIALS_FILE
# 或
export ALIBABA_CLOUD_CREDENTIALS_FILE="/path/to/credentials"
```

#### Q7: 提示 "Client is not open in the specified credentials file"

**现象**：
配置文件中找不到指定的 profile（默认为 `[default]`）。

**原因**：
配置文件中缺少与 `ALIBABA_CLOUD_PROFILE` 环境变量指定的 section 名称。

**解决方案**：

1. 确保配置文件包含对应的 section：
   ```ini
   [default]
   type = access_key
   access_key_id = xxx
   access_key_secret = xxx
   ```

2. 或设置正确的 profile 名称：
   ```bash
   export ALIBABA_CLOUD_PROFILE="production"
   ```

   配置文件中添加：
   ```ini
   [production]
   type = access_key
   access_key_id = xxx
   access_key_secret = xxx
   ```

#### Q8: 配置文件中的凭证类型相关错误

**现象**：
```
The configured client type is empty
The configured access_key_id or access_key_secret is empty
The configured role_session_name or role_arn is empty
The configured private_key_file is empty
The configured role_name is empty
```

**原因**：
配置文件中必需字段缺失或为空。

**解决方案**：

根据不同的凭证类型，确保配置完整：

**AccessKey 类型**：
```ini
[default]
type = access_key
access_key_id = YOUR_ACCESS_KEY_ID
access_key_secret = YOUR_ACCESS_KEY_SECRET
```

**RAM 角色 ARN 类型**：
```ini
[default]
type = ram_role_arn
access_key_id = YOUR_ACCESS_KEY_ID
access_key_secret = YOUR_ACCESS_KEY_SECRET
role_arn = acs:ram::123456789012****:role/YOUR_ROLE_NAME
role_session_name = session_name
```

**ECS RAM 角色类型**：
```ini
[default]
type = ecs_ram_role
role_name = YOUR_ROLE_NAME
```

**OIDC 角色 ARN 类型**：
```ini
[default]
type = oidc_role_arn
role_arn = acs:ram::123456789012****:role/YOUR_ROLE_NAME
oidc_provider_arn = acs:ram::123456789012****:oidc-provider/YOUR_OIDC_PROVIDER
oidc_token_file_path = /path/to/oidc/token
role_session_name = session_name
```

### 4.3 ECS RAM 角色相关问题

#### Q9: 提示 "Failed to connect ECS Metadata Service"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: Failed to connect ECS Metadata Service: 
System.Net.WebException: The request url is 100.100.100.200...
```

**原因**：
1. 代码未在 ECS/ECI 实例上运行
2. 实例元数据服务不可达
3. 网络配置问题（防火墙阻止访问 100.100.100.200）
4. 连接或读取超时（默认各 1 秒）

**解决方案**：

1. **确认在 ECS/ECI 上运行**：
   ```bash
   # 测试能否访问元数据服务
   curl http://100.100.100.200/latest/meta-data/
   ```

2. **检查网络配置**：
   - 确保没有防火墙规则阻止访问 100.100.100.200
   - 确认实例网络配置正常

3. **增加超时时间**（如网络较慢）：
   ```csharp
   Config config = new Config()
   {
       Type = "ecs_ram_role",
       RoleName = "YOUR_ROLE_NAME",
       ConnectTimeout = 5000,  // 5 秒连接超时
       Timeout = 10000         // 10 秒读取超时
   };
   ```

4. **如非 ECS 环境，禁用元数据检测**：
   ```bash
   export ALIBABA_CLOUD_ECS_METADATA_DISABLED=true
   ```

#### Q10: 提示 "The role name was not found in the instance"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: The role name was not found in the instance
```

**原因**：
1. ECS 实例未绑定 RAM 角色
2. 指定的角色名称不存在
3. 实例元数据服务返回 404

**解决方案**：

1. **检查实例是否绑定角色**：
   ```bash
   # 在 ECS 实例上执行
   curl http://100.100.100.200/latest/meta-data/ram/security-credentials/
   ```
   如果返回空或 404，说明实例未绑定角色。

2. **在 ECS 控制台绑定角色**：
   - 登录 ECS 控制台
   - 选择实例 → 更多 → 实例设置 → 授予/收回 RAM 角色
   - 选择已创建的 RAM 角色并绑定

3. **不指定角色名，让 SDK 自动获取**：
   ```csharp
   Config config = new Config()
   {
       Type = "ecs_ram_role"
       // 不设置 RoleName，会自动获取
   };
   ```

#### Q11: 提示 "Failed to get RAM session credentials from ECS metadata service"

**现象**：
元数据服务响应状态码非 200，或返回的 JSON 中 `Code` 字段不是 `Success`。

**原因**：
1. 元数据服务暂时不可用
2. RAM 角色配置有误
3. 角色权限不足

**解决方案**：

1. **检查元数据服务响应**：
   ```bash
   curl http://100.100.100.200/latest/meta-data/ram/security-credentials/YOUR_ROLE_NAME
   ```

2. **查看响应中的错误信息**，根据提示调整 RAM 角色配置。

3. **确认角色权限**：确保 RAM 角色拥有所需的阿里云资源访问权限。

#### Q12: 提示 "IMDS credentials is disabled"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: IMDS credentials is disabled
```

**原因**：
环境变量 `ALIBABA_CLOUD_ECS_METADATA_DISABLED` 被设置为 `true`，禁用了实例元数据服务凭证。

**解决方案**：
```bash
# 取消禁用
unset ALIBABA_CLOUD_ECS_METADATA_DISABLED
# 或
export ALIBABA_CLOUD_ECS_METADATA_DISABLED=false
```

#### Q13: 提示 "Failed to get token from ECS Metadata Service, and fallback to IMDS v1 is disabled"

**现象**：
无法获取 IMDSv2 token，且已禁用回退到 IMDSv1。

**原因**：
启用了 `DisableIMDSv1` 选项或环境变量 `ALIBABA_CLOUD_IMDSV1_DISABLED=true`，但 IMDSv2 token 获取失败。

**解决方案**：

1. **允许回退到 IMDSv1**（不推荐，安全性较低）：
   ```csharp
   Config config = new Config()
   {
       Type = "ecs_ram_role",
       DisableIMDSv1 = false
   };
   ```

2. **检查实例是否支持 IMDSv2**：
   ```bash
   curl -X PUT "http://100.100.100.200/latest/api/token" \
        -H "X-aliyun-ecs-metadata-token-ttl-seconds: 21600"
   ```

3. **查看原始错误**，可能是网络或配置问题。

### 4.4 RAM 角色 ARN 相关问题

#### Q14: STS AssumeRole 调用失败

**现象**：
使用 `ram_role_arn` 类型时，提示权限或参数错误。

**常见错误码**：
- `NoPermission`：当前 AccessKey 没有 AssumeRole 权限
- `InvalidParameter.RoleArn`：RoleArn 格式错误
- `EntityNotExist.Role`：指定的角色不存在

**解决方案**：

1. **确认 RoleArn 格式**：
   ```
   acs:ram::账户ID:role/角色名
   例如：acs:ram::123456789012****:role/MyAppRole
   ```

2. **授予 AssumeRole 权限**：
   为使用的 AccessKey 对应的 RAM 用户/角色添加以下权限策略：
   ```json
   {
     "Version": "1",
     "Statement": [
       {
         "Effect": "Allow",
         "Action": "sts:AssumeRole",
         "Resource": "acs:ram::*:role/YOUR_ROLE_NAME"
       }
     ]
   }
   ```

3. **配置角色信任策略**：
   在 RAM 控制台编辑目标角色的信任策略，允许当前账号或 RAM 用户扮演：
   ```json
   {
     "Statement": [
       {
         "Action": "sts:AssumeRole",
         "Effect": "Allow",
         "Principal": {
           "RAM": [
             "acs:ram::123456789012****:root"
           ]
         }
       }
     ],
     "Version": "1"
   }
   ```

4. **使用环境变量**（可选）：
   ```bash
   export ALIBABA_CLOUD_ROLE_ARN="acs:ram::123456789012****:role/MyRole"
   ```

#### Q15: 提示 "Session duration should be in the range of 900s - max session duration"

**现象**：
设置的 `RoleSessionExpiration` 小于 900 秒（15 分钟）。

**原因**：
STS 临时凭证的最小有效期为 900 秒，这是阿里云的安全限制。

**解决方案**：
```csharp
Config config = new Config()
{
    Type = "ram_role_arn",
    // ...其他配置
    RoleSessionExpiration = 3600  // 至少 900，最大取决于角色配置（通常 3600-43200）
};
```

或使用 Builder 模式：
```csharp
var provider = new RamRoleArnCredentialProvider.Builder()
    .DurationSeconds(3600)  // 至少 900
    // ...其他配置
    .Build();
```

### 4.5 OIDC 相关问题

#### Q16: 提示 "OIDCTokenFilePath does not exist" 或 "cannot be read"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: OIDCTokenFilePath /path/to/token does not exist.
Aliyun.Credentials.Exceptions.CredentialException: OIDCTokenFilePath /path/to/token cannot be read.
```

**原因**：
1. OIDC Token 文件路径不存在
2. 文件路径拼写错误
3. 没有文件读取权限

**解决方案**：

1. **确认文件存在**：
   ```bash
   ls -la /path/to/oidc/token
   ```

2. **在 Kubernetes（ACK）环境中**：
   OIDC Token 通常由 Kubernetes 自动挂载到 Pod，路径通常为：
   ```
   /var/run/secrets/tokens/oidc-token
   ```
   
   确认 Pod 配置中已正确挂载：
   ```yaml
   spec:
     serviceAccountName: your-service-account
     containers:
     - name: your-app
       volumeMounts:
       - name: oidc-token
         mountPath: /var/run/secrets/tokens
     volumes:
     - name: oidc-token
       projected:
         sources:
         - serviceAccountToken:
             path: oidc-token
             expirationSeconds: 3600
             audience: "sts.aliyuncs.com"
   ```

3. **设置环境变量**：
   ```bash
   export ALIBABA_CLOUD_OIDC_TOKEN_FILE="/var/run/secrets/tokens/oidc-token"
   ```

4. **检查文件权限**：
   ```bash
   chmod 644 /path/to/oidc/token
   ```

#### Q17: OIDC AssumeRoleWithOIDC 调用失败

**常见错误**：
- `AuthenticationFail.NoPermission`：OIDC 提供商 ARN 无效
- `InvalidParameter.OIDCProviderArn`：OIDCProviderArn 格式错误
- `InvalidParameter.OIDCToken`：OIDC Token 无效或已过期

**解决方案**：

1. **确认 OIDCProviderArn 格式**：
   ```
   acs:ram::账户ID:oidc-provider/提供商名称
   例如：acs:ram::123456789012****:oidc-provider/my-oidc-provider
   ```

2. **检查 OIDC Token 有效性**：
   ```bash
   # 查看 token 内容（需要 jq 工具）
   cat /path/to/token | cut -d. -f2 | base64 -d | jq
   ```
   确认 `exp`（过期时间）和 `aud`（受众）字段。

3. **配置角色信任策略**：
   在 RAM 控制台为目标角色配置 OIDC 信任策略：
   ```json
   {
     "Statement": [
       {
         "Action": "sts:AssumeRoleWithOIDC",
         "Effect": "Allow",
         "Principal": {
           "Federated": [
             "acs:ram::123456789012****:oidc-provider/your-oidc-provider"
           ]
         },
         "Condition": {
           "StringEquals": {
             "oidc:aud": "sts.aliyuncs.com"
           }
         }
       }
     ],
     "Version": "1"
   }
   ```

4. **使用环境变量**：
   ```bash
   export ALIBABA_CLOUD_ROLE_ARN="acs:ram::123456789012****:role/MyRole"
   export ALIBABA_CLOUD_OIDC_PROVIDER_ARN="acs:ram::123456789012****:oidc-provider/MyProvider"
   export ALIBABA_CLOUD_OIDC_TOKEN_FILE="/var/run/secrets/tokens/oidc-token"
   ```

#### Q18: 提示 "Security Exception: Do not have the required permission"

**现象**：
读取 OIDC Token 文件时遇到安全异常。

**原因**：
当前进程没有文件读取权限，可能是由于 SELinux、AppArmor 或文件系统权限限制。

**解决方案**：

1. **调整文件权限**：
   ```bash
   sudo chown your-user:your-group /path/to/token
   chmod 400 /path/to/token
   ```

2. **在容器环境中**：
   确保容器运行用户有权限读取挂载的 token 文件。

3. **检查 SELinux 状态**（仅 Linux）：
   ```bash
   getenforce  # 查看状态
   # 如有必要，临时禁用（仅用于调试）
   sudo setenforce 0
   ```

### 4.6 URL 凭证提供者相关问题

#### Q19: 提示 "Credentials URI must not be null or empty"

**现象**：
```
System.ArgumentNullException: Credentials URI must not be null or empty.
Parameter name: credentialsURI
```

**原因**：
使用 `credentials_uri` 类型但未提供 URI。

**解决方案**：

1. **在配置中指定**：
   ```csharp
   Config config = new Config()
   {
       Type = "credentials_uri",
       CredentialsURI = "http://your-credentials-server/credentials"
   };
   ```

2. **使用环境变量**：
   ```bash
   export ALIBABA_CLOUD_CREDENTIALS_URI="http://your-credentials-server/credentials"
   ```

#### Q20: 提示 "Credential URI is not valid"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: Credential URI is not valid.
```

**原因**：
提供的 URI 格式不正确，无法解析为有效的 URL。

**解决方案**：

确保 URI 格式正确：
```csharp
// 正确示例
"http://your-server.com/credentials"
"https://your-server.com:8080/api/v1/credentials"

// 错误示例
"invalid-url"
"htp://typo-in-protocol.com"
```

### 4.7 默认凭证提供链相关问题

#### Q21: 提示 "not found credentials"

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: not found credentials: 
[EnvironmentVariableCredentialsProvider: Environment variable accessKeyId cannot be empty, 
CLIProfileCredentialsProvider: Unable to open credentials file:..., 
ProfileCredentialsProvider: Unable to open credentials file:..., 
EcsRamRoleCredentialProvider: Failed to connect ECS Metadata Service:...]
```

**原因**：
使用默认凭证提供链（`new Client()` 或 `new Client(null)`）时，所有凭证来源都失败。

**凭证提供链顺序**：
1. 环境变量（`ALIBABA_CLOUD_ACCESS_KEY_ID` 等）
2. OIDC 角色（如果设置了相关环境变量）
3. CLI 配置文件（`~/.aliyun/config.json`）
4. 凭证配置文件（`~/.alibabacloud/credentials`）
5. ECS 实例元数据（如未禁用）
6. URL 凭证服务（如设置了 `ALIBABA_CLOUD_CREDENTIALS_URI`）

**解决方案**：

1. **选择一种凭证来源并正确配置**，例如：

   **选项 A：使用环境变量**
   ```bash
   export ALIBABA_CLOUD_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID"
   export ALIBABA_CLOUD_ACCESS_KEY_SECRET="YOUR_ACCESS_KEY_SECRET"
   ```

   **选项 B：创建配置文件**
   ```bash
   mkdir -p ~/.alibabacloud
   cat > ~/.alibabacloud/credentials << EOF
   [default]
   type = access_key
   access_key_id = YOUR_ACCESS_KEY_ID
   access_key_secret = YOUR_ACCESS_KEY_SECRET
   EOF
   ```

   **选项 C：在 ECS 上使用 RAM 角色**
   确保 ECS 实例已绑定 RAM 角色

   **选项 D：明确指定凭证类型**
   ```csharp
   // 不使用默认链，明确指定凭证类型
   Config config = new Config()
   {
       Type = "access_key",
       AccessKeyId = "YOUR_ACCESS_KEY_ID",
       AccessKeySecret = "YOUR_ACCESS_KEY_SECRET"
   };
   var client = new Client(config);
   ```

2. **调试凭证提供链**：
   
   查看错误消息中每个提供者的具体失败原因，逐一排查。

#### Q22: 提示 "Environment variable accessKeyId cannot be empty"

**现象**：
默认凭证链首先尝试从环境变量获取凭证，但变量未设置或为空。

**原因**：
环境变量 `ALIBABA_CLOUD_ACCESS_KEY_ID` 或 `ALIBABA_CLOUD_ACCESS_KEY_SECRET` 未设置。

**解决方案**：

1. **如需使用环境变量**，正确设置：
   ```bash
   export ALIBABA_CLOUD_ACCESS_KEY_ID="YOUR_ID"
   export ALIBABA_CLOUD_ACCESS_KEY_SECRET="YOUR_SECRET"
   ```

2. **如不使用环境变量**，配置其他凭证来源（配置文件、ECS 角色等），忽略此错误即可。

### 4.8 网络和超时相关问题

#### Q23: 网络请求超时

**现象**：
调用 STS 服务或元数据服务时超时。

**原因**：
1. 网络延迟较高
2. 默认超时时间过短（ECS 元数据默认 1 秒，其他默认 5-10 秒）
3. 防火墙或代理配置问题

**解决方案**：

1. **增加超时时间**：
   ```csharp
   Config config = new Config()
   {
       Type = "ram_role_arn",
       // ...其他配置
       ConnectTimeout = 10000,  // 连接超时 10 秒
       Timeout = 30000          // 读取超时 30 秒
   };
   ```

2. **配置代理**（如果需要）：
   ```csharp
   Config config = new Config()
   {
       Type = "access_key",
       // ...其他配置
       Proxy = "http://proxy-server:port"
   };
   ```

3. **检查网络连通性**：
   ```bash
   # 测试 STS 服务
   curl -v https://sts.aliyuncs.com
   
   # 测试元数据服务（仅 ECS）
   curl -v http://100.100.100.200/latest/meta-data/
   ```

#### Q24: 提示 "Http response is null"

**现象**：
HTTP 请求返回 null 响应。

**原因**：
网络请求完全失败，未收到任何响应。

**解决方案**：

1. 检查网络连接
2. 确认目标服务可访问
3. 检查 DNS 解析是否正常
4. 如在内网环境，确认是否需要配置代理或使用 VPC 端点：
   ```bash
   # 使用 VPC 端点（内网访问）
   export ALIBABA_CLOUD_VPC_ENDPOINT_ENABLED=true
   export ALIBABA_CLOUD_STS_REGION=cn-hangzhou
   ```

### 4.9 凭证刷新和缓存相关问题

#### Q25: 临时凭证过期后未自动刷新

**现象**：
使用 RAM 角色、OIDC 等临时凭证时，凭证过期后仍使用旧凭证导致鉴权失败。

**原因**：
每次调用 `GetCredential()` 都会检查凭证是否过期并自动刷新，但如果：
1. 缓存了凭证对象而非客户端
2. 异步刷新未正常工作

**解决方案**：

1. **不要缓存凭证对象**，始终通过客户端获取：
   ```csharp
   // 错误做法
   var credential = client.GetCredential();
   // 长时间使用 credential...（可能已过期）
   
   // 正确做法
   var client = new Client(config);
   // 每次使用时获取
   var credential = client.GetCredential();
   ```

2. **使用异步凭证更新**（仅 ECS RAM 角色）：
   异步更新默认已启用，每分钟检查一次凭证是否需要刷新。

3. **检查 Expiration 字段**：
   ```csharp
   var credential = client.GetCredential();
   Console.WriteLine($"Expiration: {credential.Expiration}");
   ```

### 4.10 平台兼容性相关问题

#### Q26: 在 .NET Framework 4.5 上运行异常

**现象**：
某些功能在 .NET Framework 4.5 上表现异常或报错。

**原因**：
库同时支持 .NET Framework 4.5 和 .NET Standard 2.0，某些 API 在不同平台上有细微差异。

**解决方案**：

1. **推荐升级到 .NET Core 3.1 或 .NET 6/7/8**，获得更好的性能和兼容性。

2. **如必须使用 .NET Framework 4.5**：
   - 确保已安装最新的 .NET Framework 补丁
   - 注意异步方法的行为差异
   - 部分字典操作 API 有条件编译（见测试代码中的 `#if NET45` 块）

#### Q27: 在 Linux/macOS 上路径问题

**现象**：
Windows 路径在 Linux/macOS 上无法使用。

**解决方案**：

使用跨平台的路径表示方式：
```csharp
// 使用 Path.Combine 而非硬编码路径
string configPath = Path.Combine(
    Environment.GetFolderPath(Environment.SpecialFolder.UserProfile),
    ".alibabacloud",
    "credentials"
);

// 或使用环境变量
string home = Environment.GetEnvironmentVariable("HOME") 
    ?? Environment.GetFolderPath(Environment.SpecialFolder.UserProfile);
```

### 4.11 安全最佳实践

#### Q28: 如何安全地管理 AccessKey？

**建议**：

1. **不要将 AccessKey 硬编码在代码中**：
   ```csharp
   // 不推荐
   config.AccessKeyId = "LTAI4G***********";
   
   // 推荐：使用环境变量或配置文件
   ```

2. **使用临时凭证**：
   优先使用 STS Token、RAM 角色等临时凭证，避免使用长期 AccessKey。

3. **最小权限原则**：
   为 RAM 用户/角色配置最小必需权限。

4. **定期轮换 AccessKey**：
   定期更换 AccessKey，降低泄露风险。

5. **配置文件权限**：
   ```bash
   chmod 600 ~/.alibabacloud/credentials  # 仅所有者可读写
   ```

6. **使用 RAM 角色而非 AccessKey**：
   在 ECS、容器、Kubernetes 等环境优先使用 RAM 角色。

#### Q29: 如何验证凭证是否有效？

**方法**：

```csharp
using Aliyun.Credentials;
using Aliyun.Credentials.Exceptions;

try
{
    var client = new Client();
    var credential = client.GetCredential();
    
    Console.WriteLine($"凭证类型: {credential.Type}");
    Console.WriteLine($"AccessKeyId: {credential.AccessKeyId}");
    
    if (credential.Expiration > 0)
    {
        var expireTime = DateTimeOffset.FromUnixTimeMilliseconds(credential.Expiration);
        Console.WriteLine($"过期时间: {expireTime}");
        
        if (expireTime < DateTimeOffset.Now)
        {
            Console.WriteLine("警告：凭证已过期！");
        }
    }
    
    Console.WriteLine("凭证获取成功！");
}
catch (CredentialException ex)
{
    Console.WriteLine($"凭证获取失败: {ex.Message}");
}
```

### 4.12 其他常见问题

#### Q30: 提示 "Ivalid initialization parameter" (注意：拼写错误)

**现象**：
```
Aliyun.Credentials.Exceptions.CredentialException: Ivalid initialization parameter
```

**原因**：
传递给 `Client` 构造函数的参数类型不正确。有效类型为：
- `null`（使用默认凭证提供链）
- `Config` 对象
- `IAlibabaCloudCredentialsProvider` 实现

**解决方案**：
```csharp
// 正确用法
var client1 = new Client();  // 默认链
var client2 = new Client(null);  // 默认链
var client3 = new Client(new Config { Type = "access_key", ... });  // Config
var client4 = new Client(new StaticAKCredentialsProvider("id", "secret"));  // Provider

// 错误用法
var client = new Client("some string");  // 错误！
var client = new Client(123);  // 错误！
var client = new Client(new Client());  // 错误！
```

#### Q31: 异步方法 `GetCredentialAsync()` 使用建议

**推荐用法**：
```csharp
// 在异步上下文中使用
public async Task<void> MyAsyncMethod()
{
    var client = new Client(config);
    var credential = await client.GetCredentialAsync();
    // 使用 credential...
}

// 避免在同步上下文中阻塞等待
public void MySyncMethod()
{
    var client = new Client(config);
    // 不推荐：可能导致死锁
    var credential = client.GetCredentialAsync().Result;
    
    // 推荐：使用同步方法
    var credential2 = client.GetCredential();
}
```
